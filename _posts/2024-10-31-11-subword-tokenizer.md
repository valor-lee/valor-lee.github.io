# 개요

OOV(Out-Of-Vocabulary) 또는 UNK(Unknown Token)가 발생하면 문제를 푸는 것이 까다로워집니다.

서브워드 분리(Subword segmenation) 작업은 하나의 단어는 더 작은 단위의 의미있는 여러 서브워드들(Ex. birthplace = birth + place)의 조합으로 구성된 경우가 많기 때문에,

하나의 단어를 여러 서브워드로 분리해서 단어를 인코딩 및 임베딩하겠다는 의도를 가진 전처리 작업입니다.

OOV나 희귀 단어, 신조어와 같은 문제를 완화시킬 수 있습니다.

서브워드 토크나이저의 주요 알고리즘인 바이트 페어 인코딩과 실제 실무에서 사용하는 서브워드 토크나이저 구현체인 SentencePiece와 Huggingface의 Tokenizers에 대해서 정리하겠습니다.

# 바이트 페어 인코딩(Byte Pair Encoding, BPE)

글자 단위로 분리된 상태의 초기 구성에서 연속적으로 가장 많이 등장한 글자의 쌍을 찾아서 하나의 글자로 병합하는 방식을 수행합니다.

글자(charcter) 단위에서 점차적으로 단어 집합(vocabulary)을 만들어 내는 Bottom up 방식의 접근을 사용합니다.

전체 말뭉치에서 가장 자주 등장하는 문자 쌍을 반복적으로 병합하여 새로운 서브워드를 만듭니다. 

이 방식은 특히 복잡한 형태 변화를 가진 언어(한국어, 독일어 등)에서 유용합니다.


## BPE 알고리즘의 작동 원리

### 초기화

    모든 단어를 문자 단위의 초기 토큰으로 분해합니다.

    예를 들어 "school"을 "s c h o o l"로 분리하여 각각을 개별 문자 토큰으로 간주합니다.

### 빈도 계산 및 문자 쌍 병합

    전체 데이터에서 가장 자주 등장하는 문자 쌍을 찾고 결합합니다.

    예를 들어, "s c h o o l"에서 "o o"가 가장 빈도가 높다면 이를 "oo"로 병합하여 "s c h oo l"로 만듭니다.

    이후 이 병합 과정을 반복하여 가장 자주 등장하는 문자 쌍을 새로운 토큰으로 만들어갑니다.

### 반복

    정해진 병합 횟수나 조건에 도달할 때까지, 문서에서 가장 자주 등장하는 문자 쌍을 반복적으로 병합합니다.

    최종적으로 자주 등장하는 문자 쌍이 결합되면서 서브워드 사전이 생성됩니다.

### 최종 서브워드 사전 구성

    모든 병합 작업이 완료되면, 이 서브워드들의 리스트가 BPE 사전이 됩니다. 

    이 사전을 사용하여 새로운 단어가 등장할 때도 해당 단어를 서브워드 조합으로 분해하여 OOV 문제를 해결할 수 있습니다.

## BPE의 장점

OOV 문제 해결: 새로운 단어라도 사전에 없는 단어가 서브워드 단위로 분해되므로, 미리 정의된 임베딩으로 처리 가능합니다.

희소성 문제 완화: 고정된 사전 크기를 유지하면서도 다양한 형태의 단어를 처리할 수 있어, 메모리 효율성이 높습니다.

언어 독립성: 모든 언어에 적용 가능하며, 특히 한국어, 독일어 같은 복잡한 형태 변화를 가진 언어에서 유리합니다.

## BPE의 한계

의미 분별 한계: 서브워드로 나뉘기 때문에 단어의 의미적 차이를 구별하는 데는 한계가 있습니다.

고정된 병합 규칙: 병합 규칙이 사전 학습된 데이터에 고정되므로, 특정 도메인이나 컨텍스트에서 새로운 규칙을 적용하기 어렵습니다.

# 센텐스피스(SentencePiece)

구글에서 개발한 언어 독립적인 텍스트 토크나이저입니다.

텍스트 전처리 시 필요했던 공백 분리 작업 없이 모든 언어에서 일관된 토크나이징을 할 수 있게 설계되었습니다. 

서브워드 분할 기법을 사용하며, 특히 공백마저 서브워드 토큰으로 취급하여 단어와 문장의 토큰화에 뛰어난 성능을 보입니다.

## SentencePiece의 알고리즘

### 훈련 데이터 준비

    입력 텍스트 데이터 전체를 하나의 문자 스트림으로 연결한 뒤, 모든 문자를 공백 포함 하나의 연속된 스트림으로 만듭니다.


### 서브워드 학습 (BPE 또는 Unigram)

    훈련 데이터에서 각 서브워드의 빈도나 확률을 계산하여, 서브워드 사전을 구성합니다.

    BPE 방식에서는 자주 등장하는 문자 쌍부터 병합하고, Unigram 방식에서는 각 서브워드 분할을 확률적으로 평가하여 최적의 서브워드 분할을 학습합니다.

### 토큰화

    훈련된 서브워드 사전을 바탕으로 새로운 텍스트를 서브워드로 분해합니다.

    SentencePiece는 문장에서 단어를 서브워드 단위로 나누어 필요한 만큼의 정보만 임베딩할 수 있어 효율적입니다.


## SentencePiece의 예시

다음 예시는 SentencePiece가 공백을 포함해 서브워드 단위로 토큰화하는 방식을 보여줍니다.

입력 텍스트: "I love machine learning"

훈련 데이터에 기반한 BPE 분할 결과: ["▁I", "▁love", "▁machine", "▁learning"]

위 예시에서 ▁는 공백을 의미하는 특수 기호로, SentencePiece가 공백을 구분자로 인식하지 않도록 하면서도 문맥을 반영할 수 있게 도와줍니다.

## SentencePiece의 장점

일관된 서브워드 분할: BPE와 Unigram 기반 서브워드 분할을 지원하여 다양한 언어에서 일정하게 단어를 분해할 수 있습니다.

공백 없는 언어 지원: 한국어, 중국어와 같은 언어에서도 별도의 띄어쓰기 전처리 없이 사용할 수 있습니다.

유연한 토큰 처리: SentencePiece는 텍스트 전처리 없이 모든 형태의 텍스트를 하나의 통일된 스트림으로 처리하므로, 사전 작업이 줄어듭니다.

## SentencePiece의 한계

설정 필요성: BPE와 Unigram 모델을 선택해야 하며, 최적의 사전 크기나 병합 횟수 설정이 필요합니다.

고정된 서브워드 사전: 학습된 서브워드 사전이 고정되어 있어 특정 도메인에서는 최적의 성능을 위해 추가 미세 조정이 필요할 수 있습니다.

# 서브워드텍스트인코더(SubwordTextEncoder)

TensorFlow Datasets 라이브러리를 통해 사용할 수 있는 토크나이저입니다.

단어가 포함된 특정 문장이나 단어에서 자주 쓰이는 서브워드 조합을 자동으로 생성하고,

새로운 단어나 드물게 사용되는 단어도 서브워드로 분해하여 표현할 수 있게 합니다.

## SubwordTextEncoder의 주요 특징


### 서브워드 단위의 효율적 학습

    텍스트를 전체 단어가 아닌 서브워드 단위로 분할해 처리하므로, 드문 단어가 포함된 데이터도 적은 어휘로 학습할 수 있습니다.

    특히 한국어, 중국어처럼 공백 없이 이어진 텍스트를 처리하는 데 강점을 보입니다.

### BPE와 유사한 서브워드 분할 방식

    SubwordTextEncoder는 BPE와 유사한 방식으로 서브워드를 생성합니다.

    가장 빈도가 높은 문자 쌍을 반복적으로 병합하여 서브워드 사전을 구성합니다.

    이를 통해 의미를 유지하면서 필요한 서브워드 어휘만을 선별하고 불필요한 중복을 줄일 수 있습니다.

### OOV 문제 해결

    학습되지 않은 단어가 있어도 기존 서브워드 조합을 통해 표현할 수 있어 OOV 문제를 효과적으로 해결합니다.


## SubwordTextEncoder의 한계

고정된 어휘 크기: 특정 도메인에 맞추어 어휘를 재구성하는 작업이 필요할 수 있습니다.

학습 데이터 종속성: 어휘 생성이 훈련 데이터에 따라 달라지므로, 도메인이 다르거나 특수 용어가 포함된 데이터에서는 추가 학습이 필요합니다.

# 허깅페이스 토크나이저(Huggingface Tokenizer)

**허깅페이스 토크나이저(Hugging Face Tokenizers)**는 NLP(Natural Language Processing) 작업을 위한 고성능의 토크나이저 라이브러리입니다.

다양한 토크나이징 알고리즘과 병렬 처리를 통한 빠른 토크나이징 속도를 지원합니다. 
- BERT, GPT-2, RoBERTa, T5 등 여러 트랜스포머 모델에서 사용하는 다양한 토크나이징 방식을 제공합니다.

정수 인코딩 과정을 최적화하여 효율적이고 확장 가능한 NLP 모델을 학습할 수 있게 해줍니다.

## Hugging Face Tokenizers의 주요 특징

### 다양한 토크나이징 방식 지원

    BPE(Byte Pair Encoding), WordPiece, Unigram, SentencePiece 등 여러 서브워드 토크나이징 알고리즘을 지원하여 특정 모델에 맞는 최적의 토크나이징 방식을 선택할 수 있습니다.


### 고속 병렬 처리

    Rust 언어로 구현되어 매우 빠른 성능을 제공하며, 대규모 텍스트 데이터를 효율적으로 전처리할 수 있습니다.

    멀티스레딩을 통해 대량의 데이터 처리 속도를 높일 수 있어, 대형 데이터셋이나 긴 텍스트를 다루는 작업에 최적화되어 있습니다.


### 입력 데이터 전처리 최적화

    텍스트 정규화, 클리닝, 서브워드 분할, 패딩, 트렁케이팅 등의 기능을 포함해, 모델이 요구하는 형태로 데이터를 쉽게 전처리할 수 있습니다.

### 모델 학습을 위한 어휘 학습 지원

    특정 데이터셋에 맞춰 커스텀 토크나이저를 학습할 수 있습니다.
    
    사용자 지정 어휘를 생성하여 특정 도메인에 맞는 맞춤형 토크나이저를 구축할 수 있습니다.

### 다양한 출력 형식 제공

    ID 토큰, 어텐션 마스크, 토큰 타입 ID 등을 반환합니다.
    
    이를 통해 모델 학습 및 추론에 필요한 모든 요소를 준비할 수 있습니다.

## Hugging Face Tokenizers의 구조와 동작 방식

1. Normalizer

    입력 텍스트를 정규화하여 불필요한 기호를 제거하고 일관된 형태로 변환합니다.

    예를 들어, 대문자를 소문자로 변환하거나 특정 기호를 정리하는 작업을 합니다.

2. PreTokenizer
    
    입력 텍스트를 토큰화할 준비를 합니다.
    
    공백이나 구두점 기준으로 초기 분할을 수행합니다.

3. Tokenizer
    
    주 토크나이징 알고리즘 (예: BPE, WordPiece, Unigram)을 적용해 텍스트를 서브워드 단위로 분할하고, 이 서브워드를 모델이 이해할 수 있는 정수 ID로 매핑합니다.

4. PostProcessor

    추가적인 전처리 작업을 수행합니다.
    
    예를 들어, BERT 모델에 필요한 [CLS], [SEP]와 같은 특별 토큰을 추가합니다.

5. Decoder

    토크나이징된 ID 시퀀스를 원본 텍스트로 다시 변환할 수 있습니다.
    
    이는 예측된 토큰을 디코딩하여 가독성을 유지하는 데 유용합니다.

# 기타 토크나이저

이 외 ByteLevelBPETokenizer, CharBPETokenizer, SentencePieceBPETokenizer 등이 존재하며 선택에 따라서 사용할 수 있습니다.

- BertWordPieceTokenizer : BERT에서 사용된 워드피스 토크나이저(WordPiece Tokenizer)
- CharBPETokenizer : 오리지널 BPE
- ByteLevelBPETokenizer : BPE의 바이트 레벨 버전
- SentencePieceBPETokenizer : 앞서 본 패키지 센텐스피스(SentencePiece)와 호환되는 BPE 구현체
