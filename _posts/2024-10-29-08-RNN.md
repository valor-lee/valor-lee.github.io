# 개요

피드 포워드 신경망은 입력의 길이가 고정되어 있어 자연어 처리를 위한 신경망으로는 한계가 있었습니다.

다양한 길이의 입력 시퀀스를 처리할 수 있는 인공 신경망이 필요하게 되었는데,

자연어 처리에 대표적으로 사용되는 인공 신경망이 바로 **순환 신경망(Recurrent Neural Network, RNN)**입니다.

# 순환 신경망(Recurrent Neural Network, RNN)

입력과 출력을 시퀀스 단위로 처리하는 시퀀스(Sequence) 모델입니다.

은닉층의 노드에서 활성화 함수를 통해 나온 결과값을 출력층 방향으로도 보내면서, 다시 은닉층 노드의 다음 계산의 입력으로 보내는 특징을 갖고있습니다.

## RNN의 주요 특징

### 순환 구조

RNN의 각 셀은 이전 셀의 출력(은닉 상태)을 다음 셀로 전달하는 순환 구조를 가지고 있습니다.

이로 인해 RNN은 입력 데이터의 이전 시점 정보를 기억하고 이를 현재의 계산에 사용할 수 있습니다.

각 시점 t에서의 은닉 상태 h_t는 

`h_t = f(W * x_t + U * h_{t-1} + b)`로 표현됩니다. 

    x_t는 현재 시점의 입력 벡터,

    h_{t-1}는 이전 시점의 은닉 상태,

    W, U, b는 학습된 가중치와 편향,

    f는 활성화 함수입니다.

### 은닉 상태 (Hidden State)

RNN의 은닉 상태는 이전의 계산된 정보를 저장하고, 순환 구조를 통해 시간이 지남에 따라 업데이트됩니다.

이 은닉 상태 덕분에, RNN은 이전 시점의 정보를 현재 시점에 반영하여 문맥을 이해할 수 있습니다. 

예를 들어, 문장 내에서 문맥에 따라 단어의 의미를 해석하는 데 유리합니다.


### 출력

은닉층의 출력(은닉 상태)을 이용해 다음 단어 또는 현재 시점의 출력을 계산합니다.

보통 출력층은 소프트맥스(softmax) 함수와 결합하여 다음 단어의 확률 분포를 구하거나, 각 시점의 예측 결과를 도출합니다.

만약 전체 시퀀스의 마지막에 출력이 필요한 경우에는 마지막 은닉 상태만 사용하여 최종 출력을 예측합니다.

메모리 셀이 모든 시점(time step)에 대해서 은닉 상태값을 출력하며, 모든 시점의 은닉 상태를 전달하도록 하면, 다음층에 RNN 은닉층이 하나 더 있는 경우이거나 다 대 다(many-to-many) 문제를 풀 수 있습니다.

메모리 셀이 하나의 은닉 상태값만 전달하도록 하면 다 대 일(many-to-one) 문제를 풀 수 있습니다.
- 하나의 값은 마지막 시점(time step)의 메모리 셀의 은닉 상태값입니다.


## 기울기 소실 문제와 개선 구조

RNN의 은닉 상태가 시간에 따라 계속 곱해지면서 장기적인 문맥을 기억하기가 어려워지고, 기울기 소실 문제가 발생할 수 있습니다. 

가장 중요한 정보가 시점의 앞쪽에 위치할 경우 전체 정보에 대한 영향력이 낮아지는 **장기 의존성 문제(the problem of Long-Term Dependencies)**가 발생할 수 있습니다.


이를 개선하기 위해 **LSTM(Long Short-Term Memory)**과 **GRU(Gated Recurrent Unit)**가 고안되었습니다.

이들은 **추가적인 게이트 구조를 사용**해 중요 정보는 장기간 유지하고 불필요한 정보는 잊게 하여, 더 긴 문맥 정보를 학습할 수 있도록 합니다.


## RNN 종류

1. 깊은 순환 신경망(Deep Recurrent Neural Network)
2. 양방향 순환 신경망(Bidirectional Recurrent Neural Network)


# 장단기 메모리(Long Short-Term Memory, LSTM)

은닉층의 메모리 셀에 입력 게이트, 망각 게이트, 출력 게이트를 추가하여 불필요한 기억을 지우고, 기억해야할 것들을 정합니다.
- 즉, LSTM은 은닉 상태(hidden state)를 계산하는 식이 전통적인 RNN보다 조금 더 복잡해졌으며 셀 상태(cell state)라는 값을 추가하였습니다.

LSTM의 동작 과정
1. 포겟 게이트는 이전 셀 상태에서 잊을 정보를 결정합니다.
2. 입력 게이트는 새로 추가할 정보를 셀 상태에 추가합니다.
3. 셀 상태는 포겟 게이트와 입력 게이트의 결과를 반영하여 갱신됩니다.
4. 출력 게이트는 갱신된 셀 상태를 바탕으로 은닉 상태를 생성하여 다음 시점으로 전달합니다.


# 게이트 순환 유닛(Gated Recurrent Unit, GRU)

단순한 구조와 적은 파라미터 수를 통해 더 빠르게 학습하는 동시에 장기 의존성 문제를 해결하도록 설계된 순환 신경망입니다.

데이터 양이 적을 때는 매개 변수의 양이 적은 GRU가 조금 더 낫고, 데이터 양이 더 많으면 LSTM이 더 낫다고도 합니다.

## GRU의 기본 동작

GRU는 **업데이트 게이트(Update Gate)**와 **리셋 게이트(Reset Gate)**라는 두 가지 게이트를 통해 이전 시점의 정보를 얼마나 유지할지 결정합니다. 

LSTM과 달리 별도의 셀 상태(cell state)가 없고, 은닉 상태만을 사용하여 정보를 관리합니다. 

이를 통해 GRU는 더 간단하게 정보를 전달하고 학습합니다.
