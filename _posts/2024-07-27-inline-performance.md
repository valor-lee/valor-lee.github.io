# 인라인(inline) 함수란,
> 정의한 코드들이 호출된 자리에 복사하도록 컴파일러에게 제안하는 함수를 의미합니다.

* 제안한다는 말은, 인라인 함수 지정자를 컴파일러의 판단에 따라 복사할 수도 안할 수도 있다는 뜻입니다.


## 사용법

```c
inline int add(int i, int j) { return i + j; }
```

* 위와 같이 main을 제외한 모든 함수를 inline 함수 지정자를 사용하여 인라인으로 선언 및 정의할 수 있습니다.
* 또한, 정적 로컬 변수는 inline 함수 내에 정의할 수 없습니다.

# 인라인 함수 장단점

## 장점

### 1. 매크로의 한계 극복

**매크로 함수의 단점**은

> 1. 정의 내용이 길어질 수록 가독성이 낮아집니다.
> 2. gdb나 cgdb로 디버깅이 거의 불가능합니다.
> 3. 매크로 함수는 전처리기를 통해 코드 자체가 복사되기 때문에 문법 오류가 발생할 수 있습니다.
