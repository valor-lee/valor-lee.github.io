# 개요

# 지도학습과 비지도 학습

지도학습은 레이블이라는 정답과 함께 학습하는 것을 말합니다.
- 자연어 처리는 지도학습에 속합니다.

비지도 학습은 데이터에 레이블이 없는 학습을 말합니다.
- 텍스트 처리 분야의 토픽 모델링 알고리즘인 LSA나 LDA는 비지도 학습에 속합니다.

자기지도 학습은 레이블이 없는 데이터가 주어지면 모델이 학습을 위해서 스스로 데이터로부터 레이블을 만들어서 학습하는 경우를 말합니다.
- Word2Vec과 같은 워드 임베딩 알고리즘이나, BERT와 같은 언어 모델의 학습 방법을 들 수 있습니다.

# 혼동 행렬(Confusion Matrix)

정확도 지표만으론 모델의 성능을 정확히 표현하지 못합니다.

그래서 맞춘 결과과 틀린 결과에 대한 세부 내용을 파악하기 위해 혼동 행렬을 사용합니다.

| | 예측 참 | 예측 거짓|
| -- |-- |-- |
| 실제 참 | TP | FN |
| 실제 거짓 | FP | TN |

위 행렬을 이용하여 정밀도(precision), 재현율(recall)을 계산합니다.
- 두 지표는 실제 True인 정답을 모델이 True로 예측한 경우(TP)에 관심이 있습니다.

## 정밀도

`Precision = TP / (TP + FP)`

모델이 True라고 분류한 것 중에 실제 True인 비율입니다.

## 재현율

`recall = TP / (TP + FN)`

실제 참인 것인 것들 중에 모델이 True로 판단한 비율입니다.

## 정확도

`accuracy = (TP + TN) / (TP + TN + FP + FN)`

전체 예측한 데이터 중에서 정답을 맞춘 것에 대한 비율입니다.


# 과적합(Overfittin) / 과소적합(Underfitting)

**과적합**

훈련 데이터를 과하게 학습한 경우를 의미합니다.

이때는 훈련데이터에서 오차는 적지만, 테스트 데이터에서 오차가 크게 발생합니다.

- 에포크(전체 훈련 데이터에 대한 훈련 횟수)가 커질수록 테스트 데이터의 오차가 커지게 됩니다.

이때, 테스트 데이터나 실제 서비스에서 정확도가 좋지 않는 현상이 발생합니다.


**과소적합**

테스트 데이터의 성능이 올라갈 여지가 있음에도 훈련을 덜 한 상태를 의미합니다.

## 테스트 데이터의 분리

과적합 / 과소 적합을 판단하기 위해 테스트 데이터를 사용해야 합니다.

현업에서는 테스트 데이터를

1. 과적합 모니터링, 하이퍼파라미터 튜닝을 위한 검증 데이터
2. 성능 평가를 위한 테스트 데이터


두 가지 용도로 분리해야 합니다.

## 일반적인 딥러닝 학습과정

1. 주어진 데이터를 훈련 데이터, 검증 데이터, 테스트 데이터로 나눈다.
   - 6:2:2
2. 훈련 데이터로 모델을 학습한다. (에포크 + 1)
3. 검증 데이터로 모델을 평가하여 검증 데이터에 대한 정확도와 오차를 계산한다.
4. 검증 데이터의 오차가 증가하였다면 과적합 징후이므로 학습 종료하고,하고 아니라면 2단계로 이동한다.
5. 모델의 학습이 종료되었으니 테스트 데이터로 모델으 평가한다.


## 과적합을 막는 방법

드롭 아웃(Dropout), 조기 종료(Early Stopping)과 같은 몇 가지 방법이 존재합니다.

# 선형 회귀(Linear Regression)

독립 변수에 대한 종속 변수의 관계를 정의하여 독립 변수에 따른 종속 변수의 값을 예측하는 모델입니다.


## 비용함수

실제값과 예측값에 대한 오차에 대한 식을 의미합니다.

목적 함수(Objective function)는 함수의 값을 최소화 또는 최대화하는 목적을 가진 함수를 일컫습니다.

여기서 비용 함수(Cost function) 또는 손실 함수(Loss function)는 함수의 값을 최소화하는 목적을 가진 함수를 일컫습니다.

회귀 문제에서는 보통 **평균 제곱 오차(Mean Squared Error, MSE)**를 사용합니다.\



## 옵티마이저(Optimizer) : 경사하강법(Gradient Descent)

선형 회귀를 포함한 수많은 머신 러닝, 딥 러닝의 학습은 결국 비용 함수를 최소화하는 매개 변수인 와 을 찾기 위한 작업을 수행합니다. 

이때 사용되는 알고리즘을 옵티마이저(Optimizer) 또는 최적화 알고리즘이라고 부릅니다.

여기서는 가장 기본적인 옵티마이저 알고리즘인 경사 하강법(Gradient Descent)에 대해서 정리하겠습니다.


경사 하강법(Gradient Descent)은 최적화 알고리즘의 한 종류로, 주로 머신러닝 모델의 파라미터를 최적화하는 데 사용됩니다. 기본적인 원리는 손실 함수(loss function)의 값을 최소화하는 방향으로 파라미터를 업데이트하는 것입니다.

1. **손실 함수**: 모델의 예측값과 실제값의 차이를 측정하는 함수입니다. 이 값이 작을수록 모델의 성능이 좋다고 할 수 있습니다.

2. **기울기 계산**: 현재 파라미터 값에서 손실 함수의 기울기(gradient)를 계산합니다. 기울기는 손실 함수의 변화율을 나타내며, 이 방향이 손실 함수를 최소화하는 방향입니다.

3. **파라미터 업데이트**: 기울기를 이용해 파라미터를 조정합니다.
   - `파라미터 = 파라미터 - 학습률 * 기울기`

4. **반복**: 이 과정을 여러 번 반복하여 손실 함수를 최소화하는 최적의 파라미터를 찾습니다.

경사 하강법은 간단하고 직관적이지만, 지역 최적점(local minimum)에 빠질 수 있는 단점이 있습니다. 다양한 변형(예: 미니배치 경사 하강법, 모멘텀, Adam 등)을 통해 이러한 문제를 해결하기도 합니다.

# 로지스틱 회귀(Logistic Regression)

이진 분류 문제를 풀기 위한 대표적인 알고리즘입니다.

이진 분류를 풀기 위한 알고리즘으로 선형 회귀를 적절하지 않습니다. 직선을 사용해야 하기 때문입니다.

이 문제를 해결하기 위해 출력이 0 ~ 1 사이 값을 가지면서 곡선 형태로 그려질 수 있는 시그모이드 함수를 사용합니다.

## 시그모이드 함수(Sigmoid Function)

딥러닝 알고리즘이 주어진 데이터에 대한 가장 적합한 가중치와 편향을 구하게 됩니다.


## 로지스틱 회귀의 비용 함수

평균 제곱 오차를 비용함수로 사용하면 로컬 미니멈에 빠지기 쉽습니다.

그래서 크로스 엔트로비 함수를 사용합니다.

가중치를 찾기 위해 크로스 엔트로비 함수의 평균을 취한 함수를 사용합니다.

# 소프트맥스 회귀(Softmax Regression)

3개 이상의 선택지 중에 1개를 고르는 다중 클래스 분류 문제를 위한 방법론입니다.

선택지의 총 개수를 k라고 할 때, k차원의 벡터를 입력받아 각 클래스에 대한 확률을 추정합니다.

k차원의 벡터를 입력받아서 모든 벡터 원소의 값을 0과 1사이의 값으로 값을 변경하여 다시 k차원의 벡터를 반환합니다.

입력 백터가 차원이 커지면 가중치를 곱해서 차원을 낮출 수 있습니다.
