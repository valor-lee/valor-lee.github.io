# 개요


자연어 처리에서 텍스트를 표현하는 방법으로는 여러가지 방법이 있습니다.

그 중 정보 검색과 텍스트 마이닝 분야에서 주로 사용되는 카운트 기반의 텍스트 표현 방법인 

**DTM(Document Term Matrix)과 TF-IDF(Term Frequency-Inverse Document Frequency)**

에 대해서 정리하겠습니다.

위 방법을 통해 텍스트를 수치화하면,

SLM을 통해 여러 문서로 이루어진 텍스트 데이터가 있을 때 어떤 단어가 특정 문서 내에서 얼마나 중요한 것인지를 나타내거나

문서의 핵심어 추출, 

검색엔진에서 검색 결과의 순위 결정, 

문서들 간의 유사도를 구하는 등의 용도로 사용할 수 있습니다.


# 단어의 표현 방법

![word-representation](https://wikidocs.net/images/page/31767/wordrepresentation.PNG)

<h7>출처 : https://wikidocs.net/31767</h7>

국소 표현(Local Representation)과 분산 표현(Distributed Representation) 방법으로 나뉩니다.

## 국소 표현법

해당 단어 그자체만 보고, 특정값을 맵핑하는 단어를 표현하는 방법입니다.

이산 표현(Discrete Representation)이라고도 합니다.



## 분산 표현법

함께 자주 사용되는 단어로 정의하게 됩니다.

이 덕분에 늬앙스를 표현할 수 있습니다.

연속 표현(Continuous Representation)이라고도 불립니다.

# Bag of Words(BoW)

단어의 등장 순서를 고려하지 않고 빈도수 기반의 단어 표현 방법입니다.

아래의 과정을 거칩니다.

1.  각 단어에 고유한 정수 인덱스를 부여합니다. 
2.  각 인덱스의 위치에 단어 토큰의 등장 횟수를 기록한 벡터를 만듭니다.


BoW는 어떤 단어가 얼마나 등장했는지를 기준으로 문서가 어떤 성격의 문서인지 판단하는 작업에 사용됩니다.

- 분류 문제나 여러 문서간의 유사도를 구하는 문제에 사용됩니다.


# 문서 단어 행렬(Document-Term Matrix, DTM)

서로 다른 문서들의 BoW들을 결합한 표현 방법입니다.

문서들을 서로 비교할 수 있도록 수치화할 수 있다는 점에 의의가 있습니다.

## DTM 한계

문서의 크기가 커질수록 공간적 낭비와 계산 리소스를 증가시킬 수 있다는 단점이 있습니다.

더구나, 대부분의 값이 0인 희소 벡터 또는 희소 행렬이 많아질 수 있습니다.

# TF-IDF(Term Frequency-Inverse Document Frequency)

단어 빈도-역 문서 빈도는 DTM 내의 각 단어들마다 중요한 정도를 가중치로 주는 방법입니다.

주로 

문서의 유사도를 구하는 작업, 

검색 시스템에서 검색 결과의 중요도를 정하는 작업,

문서 내에서 특정 단어의 중요도를 구하는 작업

에 쓰일 수 있습니다.

## 절차

문서를 d, 단어를 t, 문서의 총 개수를 n이라고 표현할 때 TF, DF, IDF는 각각 다음과 같이 정의할 수 있습니다.

1. tf(d,t) : 특정 문서 d에서의 특정 단어 t의 등장 횟수
2. df(t) : 특정 단어 t가 등장한 문서의 수
3. idf(t) : df(t)에 반비례하는 수
  - `log(n/(1 + df(t)))`

모든 문서에서 자주 등장하는 는 중요도가 낮다고(TF-IDF 값이 낮으면) 판단하고,

특정 문서에서만 자주 등장하는 단어는 중요도가 높다고(TF-IDF 값이 높다면) 판단합니다.
