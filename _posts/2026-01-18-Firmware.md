# 개요

firmware 개발을 시작하기 위한 기초 정보와 펌웨어 개발자에 대해서 조사한 정보를  정리함

---

# 1. 펌웨어(Firmware)란?

특수한 목적을 달성하기 위한 하드웨어(장치, 디바이스, 장비 등)을 제어하기 위한 소프트웨어

소형 컴퓨터인 MCU(MicroController Unit)에서 동작하는 소프트웨어

MCU를 중심으로 구성된 하드웨어를 동작시키는 역할

# 2. 펌웨어 개발 분류

펌웨어 개발은 제어 대상이 어떤 것이냐에 따라 분류됨

간단한 센서와 동작을 제어하는 것에서부터 수십 개의 센서와 장치로 구성된 하드웨어를 제어해야 할 때,

OS없이 구현이 가능한 펌웨어 개발에서 고사양 MCU 또는, MPU, CPU 사용을 고려해야 하는 펌웨어 개발까지 확장될 수 있음


언어는 C가 보편적이며, C++/ Rust가 시도되고 있지만 보편화되지 않음

리눅스 기반 펌웨어 개발은 C++이 사용되기도 하지만, RCOS나 Baremental 펌웨어 개발에서는 잘 사용하지 않음
- `Baremental 펌웨어` : OS가 없는 펌웨어

| | | |
| --- | --- | --- |
| processor | MCU | CPU, MPU, SOC|
| 언어 | C | C, C++ |
| OS | 없음, RTOS | Linux, Windows CE 등 임베디드 운영체제 |
| 주변기기 제어 | 직접구현, SDK 활용 | Kernel, SDK 활용 |


MCU, MPU의 주변장치를 제어하는 소프트웨어를 디바이스 드라이버라고 하는데, 보통 IC제조사가 SDK(Software Development Kit)로 제공함

사용자가 원하는 기능을 제공하지 않는 경우 직접 구현해야 함
- **SDK를 직접 구현하기 위해서는 하드웨어에 대한 전반적인 시작과 MCU에 대한 기초지식이 바탕되어야 함**

# 3. 펌웨어 개발 순서

가장 먼저 어떤 장치를 만들지 결정해야 함
- 개발자의 입장에서만 생각해보면 MCU, MPU, SOC를 선정해야 함

다음 OS를 선정해야 함
- MCU : RTOS 사용 여부 결정
- MPU, SOC : 리눅스는 필수적으로 사용됨

다음 개발환경 구축해야 함(컴파일러를 선정한다는 의미)
- 상용 컴파일러(IAR, Keil)를 구매해서 사용
- 제조사에서 제공하는 컴파일러를 사용
- GCC를 다운로드하여 직접 컴파일
