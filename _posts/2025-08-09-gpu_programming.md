## 1. CPU(Central Processing Unit) 태생적 특징과 한계

- CPU(중앙처리장치)는 범용 연산을 잘하도록 설계되어 있음
- 구조: 
  - 적은 수의 강력한 코어(수 개~수십 개) + 높은 클럭 속도.
  - 제어 유닛(Control), 캐시에 많은 공간 할당
- 장점: 
  - 복잡하고 다양한 명령 처리 가능, 분기 예측 기법을 통한 분기(branch)와 조건문에 강함.
  - 불규칙한 메모리 접근 또는 흐름을 가지는 연산에 높은 성능 
- 한계: 
  - 병렬 처리에 최적화되어 있지 않음 → 같은 순간에 많은 연산을 동시에 수행하기 어려움.

> 예를 들어, 이미지 속 수백만 픽셀 각각을 동시에 계산해야 한다면 CPU는 한 번에 몇 개씩만 처리하므로 시간이 오래 걸립니다.

## 2. CPU 성능 향상의 물리적 한계

과거(1970~2000년대 초)에는 무어의 법칙과 클럭 속도 증가 덕분에 CPU 성능이 꾸준히 올랐지만:

- 발열 – 클럭 속도를 더 높이면 발열이 기하급수적으로 증가.
- 전력 소모 – 전력 효율이 떨어져 모바일·서버 환경에 불리.
- 병렬성 부족 – 특정 작업(그래픽·행렬 계산 등)에서는 구조적 병목. sequential 연산을 하기 때문
- 코어의 크기, 제어장치, 캐시 공간 할당의 이유로 많은 수의 연산 코어를 넣기 힘듦.


## GPU(Graphic Processing Unit)란

- 그래픽 연산 특화 장치
- 컴퓨터에 저장되는 그래픽 정보(공간, 이미지)는 눈으로 식별하기 어려운 형태로 저장되는데 이를 시각화하는 과정이 필요한데, 이를 **그래픽스 연산**이라고 함.
- 그래픽스 파이프라인 연산 가속을 위해 개발됨.
  - 정보를 화면에 나타낸다는 것은 화면을 구성하는 픽셀의 색상값을 정하는 것이고, 이 과정은 **그래픽스 파이프라인(Graphics pipline)**을 통해 이루어짐
  - 디스플레이 장치의 해상도가 발전함에 따라 연산량이 많아짐
  - 픽셀은 정보를 다른 픽셀들과 독립적으로 처리할 수 있기 때문에 병렬 그래픽스 연산이 가능함.
  - 이러한 작업을 전담할 장치가 필요하게 됨
- 구조 
  - 파이프라인 연산을 동시에 처리하기 위해 GPU 내에 코어 수를 늘리는 데에 집중함.
    - 꼭 필요한 제어 기능만 넣고 크기를 줄여 코어 수를 늘림.
  - 캐시에 상대적으로 작은 공간을 할당.
    - 그래픽스 파이프라인 연산흐름은 규칙적인 만큼 메모리에 대한 접근 형태도 대체로 규칙적이기 때문.
- 특징
  - 높은 대역폭을 갖는 고성능 메모리가 필요함.
    - 많은 수의 코어에 동시에 데이터를 공급해야 하기 때문.
  - FLOPS 관점에서 CPU 대비 높은 처리 성능(연산 및 메모리 접근이 규칙적일 때)


| 구분 | CPU (Central Processing Unit) | GPU (Graphics Processing Unit) |
| --- | --- | --- |
| **발전 방향** | 각 코어의 성능 향상 | 병렬 처리(코어 수) 증가 |
| **주요 역할** | 범용 연산 처리, OS 실행, 로직·분기 처리 | 대규모 병렬 연산, 그래픽·행렬 연산 |
| **구조 특징** | SISD, MIMD | SIMT |
| **구조 특징** | 소수의 고성능 코어(보통 4~32개) + 복잡한 제어 로직 | 수백~수천 개의 단순 코어 + 단순 제어 로직 |
| **코어 성능** | 개별 코어 성능이 매우 높음 (고클럭·고IPC) | 개별 코어 성능은 낮음, 대신 병렬 수가 많음 |
| **처리 방식** | 직렬(Sequential) 처리 최적화 | 병렬(Parallel) 처리 최적화 |
| **메모리 크기** | 수십GB~1TB  | ~수십 GB |
| **메모리 접근** | 메모리 접근 지연 시간을 최적화 | 높은 메모리 대역폭 확보에 집중 |
| **장점** | \- 불규칙하고 다양한 작업 및 메모리 접근 패턴에 잘 대응  <br>\- 복잡한 로직·분기 처리 강점   <br>\- 낮은 지연(latency) | \- 대규모 연산에 매우 빠름  <br> \- 부동소수점(FLOPS)·벡터 연산 최적화   <br>\- AI·그래픽·시뮬레이션에서 뛰어남 |
| **단점** | \- 대규모 병렬 연산 속도 한계   <br>\- 코어 수 확장 비용·전력 증가<br>\- FLOPS관점에서 GPU 대비 낮은 성능 | \- 불규칙한 흐름, 복잡한 조건 분기 처리에 약하며 메모리 접근 성능 저하  <br>\- 단일 스레드 성능 낮음  <br> \- CPU 의존성이 큼  |
| **대표 활용** | 운영체제, 애플리케이션 로직, 데이터베이스, 일반 연산 | 그래픽 렌더링, AI 딥러닝, 과학 계산, 대규모 데이터 병렬 처리 |



# GPGPU, GPU 프로그래밍

과거의 GPU는 **고정된 그래픽스 파이프라인**을 통해 그래픽스 연산을 수행했음.

GPU가 발전하면서 **그래픽스 파이프라인**을 수정할 수 있는 기능이 추가되기 시작.

이 기능이 추가되면서 CPU에서만 가능하던 연산을 GPU에서도 수행할 수 있게 됨.

이처럼 **CPU가 처리하던 연산을 GPU로 수행하는 것**을 `General Purpose GPU, GPGPU`라고 함.

## 여러 분야에서 GPU를 도입하는 이유

GPU의 연산 능력이 CPU에 비해 매우 높기 때문.

GPU는 비교적 쉬운 연산, 병렬처리가 가능한 연산에서 매우 높은 처리 능력을 보임.

특히 FLOPS(FLoating Operations Per Second) 지표를 기준으로 비교할 떄 GPU의 FLOPS 성능이 크게 차이남.

- 딥러닝 분야 특성상 빠른 행렬 연산이 중요한데, 이때 FLOPS 성능이 좋은 GPU가 많이 사용됨.
- GPGPU를 활용하면 시간이 많이 소요되는 연산을 병렬처리할 수 있어서 연산 성능 및 효율을 향상시킬 수 있음.

# CUDA(Compute Unified Device Architecture)

엔비디아에서 GPU를 활용해 프로그래밍할 수 있도록 제공하는 인터페이스임.

C/C++, Java, Python 등 여러 언어에서 GPU를 제어하는 프로그래밍 가능.

CPU 중심의 프로그래밍에 GPU 연산 능력을 추가함으로써 연산 능력의 한계를 극복하고 연산 효율을 끌어올릴 수 있음.
